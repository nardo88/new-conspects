Разбор

input type="file"

input элемент с атрибутом type="file" позволяет пользователю выбрать один файл или более из файлового хранилища своего устройства. После выбора эти файлы могут быть загружены на сервер при помощи формы, или обработаны JavaScript и File API.

Свойства:

value - содержит DOMString, который представляет путь к выбранным файлам. Если пользователь выбрал несколько файлов, value представляет первый файл из списка. Отстальные файлы можно определить используя HTMLInputElement.files свойство элемента input.

files - Объект FileList, в котором перечислены все выбранные файлы. В этом списке не более одного члена, если не указан атрибут multiple.

name - имя выбранного файла

lastModified - Число, указывающее дату и время последнего изменения файла в миллисекундах с эпохи UNIX (1 января 1970 г., полночь).

size - Размер файла в байтах.

type - тип файла

События

change - срабатывает во время выбора файла

input - срабатывает во время выбора файла

FileReader

Объект FileReader позволяет веб-приложениям асинхронно читать содержимое файлов (или буферы данных), хранящиеся на компьютере пользователя, используя объекты File или Blob, с помощью которых задается файл или данные для чтения.

Свойства

error - представляет собой ошибку, происходящую при чтении файла.

result - Данные из файла. Значение свойства действительно только когда завершится операция чтения, причем формат данных зависит от способа, с помощью которого процесс был вызван.

Методы

abort() - Отмена операции чтения.

readAsArrayBuffer() - Запускает процесс чтения данных указанного Blob, по завершении, аттрибут result будет содержать данные файла в виде ArrayBuffer.

readAsBinaryString() - Запускает процесс чтения данных указанного Blob, по завершении, аттрибут result будет содержать бинарные данные файла в виде строки.

readAsDataURL() - Запускает процесс чтения данных указанного Blob, по завершении, аттрибут result будет содержать данные файла в виде data: URL.

readAsText() - Запускает процесс чтения данных указанного Blob, по завершении, аттрибут result будет содержать данные файла в виде текста.

Обработка событий

FileReader.onabort - Обработчик для события abort. Это событие срабатывает каждый раз, когда прерывается операция чтения.

FileReader.onerror - Это событие срабатывает каждый раз, когда при чтении возникает ошибка.

FileReader.onload - Это событие срабатывает при каждом успешном завершении операции чтения.

FileReader.onloadstart - Это событие срабатывает каждый раз, при запуске процесса чтения.

FileReader.onloadend - Это событие срабатывает каждый раз по окончании процесса чтения (не важно, успешном или нет).

FileReader.onprogress - Это событие срабатывает во время чтения данных из Blob.

на все перечисленные события можно подписаться, используя метод addEventListener.
