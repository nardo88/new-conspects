Модульная система при работе со стилями в React

Создавая множество компонентов, мы можем для удобства к каждой компоненте создавать и подключать собственный файл CSS. Допустим что над одним приложением работает команда из нескольких человек. Каждый работает над отдельным компонентом, и наименование классов придумывает сам. Что будет если два разработчика создадут классы с одинаковыми именами? Так как React в итоге создает единый файл JS который подключает все файлы стилей в одном месте, то стили классов с одинаковыми именам могут перебивать друг друга. Что бы этого избежать используют модульную систему классов. Как это выглядит?

Во-первых мы должны переименовать файлы CSS добавив к ним слово module
